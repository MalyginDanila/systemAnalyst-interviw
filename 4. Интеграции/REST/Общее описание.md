Определение: Архитектурный стиль взаимодействия компонентов веб приложения в сети. Указывает - как именно компоненты (FE, BE, DB) должны взаимодействовать.

Почему используют REST:
	Он обеспечивает простой, стандартизированный и масштабируемый способ создания веб-сервисов. Его простота и использование знакомых методов HTTP и встроенных механизмов кеширования делают его легким для изучения и внедрения разработчиками. В результате REST стал популярным выбором разработчиков при проектировании API для современных веб-приложений.
	
	Когда сервис построен по принципам REST и находится на взрослом уровне модели Ричардсона, его могут называть RESTful API сервис. Но не все придерживаются одного взгляда на эту позицию, как и в целом на категоризацию API веб-сервисов.

	* Простота
	* Гибкость
	* Надёжность
	* Производительность
