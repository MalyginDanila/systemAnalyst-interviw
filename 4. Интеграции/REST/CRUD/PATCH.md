Предназначен для частичного изменения какого либо ресурса

* Свойства: Не идемпотентный, не безопасный(т.к влияет на сервер и изменяет информацию), не кэшируемый
* Запрос: PATCH api.com/users/123/files/123 + тело запроса(помещается информация, которую необходимо изменить)
* Ответ: Возвращаются Тело(вернётся весь файл с новыми изменениями), заголовки, код состояния

- тело запроса должно содержать только изменяемые данные ресурса, а не все его данные. В итоге в ресурс добавляются данные, при этом ресурс не перезаписывается с нуля.
- в теле ответа обычно возвращают весь ресурс в новом виде, с учетом новых добавленных данных.
- мы можем искусственно сделать PATCH идемпотентным
